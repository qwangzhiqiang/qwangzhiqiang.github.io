git版本同步

git
创建时间:2018-10-16 19:46
阅读:
  1. 1.创建项目
  2. 将创建的本地分支同步到github中

1.创建项目

  1. 首先创建一个空文件夹或者当前项目的文件夹中在该文件夹中初始化git命令git init
    git init
    Initialized empty Git repository in C:/Users/Administrator/Desktop/ceshi/.git/
    通过该命令创建一个本地git仓库成功

  2. 将项目文件放到该文件下，通过git add 将文件添加到暂存区

    ![在这里插入图片描述](https://img-blog.csdnimg.cn/20190607232824534.png)
提示：git add * 将该文件夹下的所有文件全部添加到暂存区 
git add <file> 也可以通告该命令单独将某个文件存到暂存区

  3. 通过git commit 将文件提交到当前的分支下

    简单解释一下git commit命令，-m后面输入的是本次提交的说明，可以输入任意内容，当然最好是有意义的，这样你就能从历史记录里方便地找到改动记录。
    嫌麻烦不想输入-m “xxx”行不行？确实有办法可以这么干，但是强烈不建议你这么干，因为输入说明对自己对别人阅读都很重要。实在不想输入说明的童鞋请自行Google，我不告诉你这个参数。
    git commit命令执行成功后会告诉你，1 file changed：1个文件被改动（我们新添加的123.txt文件）；2 insertions：插入了两行内容（123.txt有两行内容）。

    将创建的本地分支同步到github中

    1.登陆github,创建一个新的仓库github地址





    远程连接库成功。
    请千万注意，把上面的michaelliao替换成你自己的GitHub账户名，否则，你在本地关联的就是我的远程库，关联没有问题，但是你以后推送是推不上去的，因为你的SSH Key公钥不在我的账户列表中。
    添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。
    把本地库的内容推送到远程，用git push命令，实际上是把当前分支master推送到远程。
    由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。
    之后通过git push origin master就可以将本地提交的代码同步到github中去了。
